合并和变基的区别

两者都可以实现代码的更新操作

合并：有两个人同时修改一个代码，最后实现两个改的东西都更新

1. 如果两个人改的部分不一样：直接合并
1. 如果两个人改的部分一样：需要手动合并部分，再实现git add . +git commit -m ...

变基：也有两个人同时修改一个代码，但是变基是A在B的基础上，以B为起点开始重新修改一个仓库

总的来说，用合并比较多
